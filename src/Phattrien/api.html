API Contract Documentation

API Overview

This document outlines the API endpoints, their request formats, and expected responses.

GET /users

Retrieve a list of all users.

Response:

{
    "users": [
        {
            "id": 1,
            "name": "John Doe",
            "email": "john.doe@example.com"
        },
        {
            "id": 2,
            "name": "Jane Smith",
            "email": "jane.smith@example.com"
        }
    ]
}

POST /users

Create a new user.

Request Body:

{
    "name": "New User",
    "email": "new.user@example.com"
}

Response:

{
    "id": 3,
    "name": "New User",
    "email": "new.user@example.com"
}

PUT /users/{id}

Update an existing user.

Request Body:

{
    "name": "Updated Name",
    "email": "updated.email@example.com"
}

Response:

{
    "id": 1,
    "name": "Updated Name",
    "email": "updated.email@example.com"
}
